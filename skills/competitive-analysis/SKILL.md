# Competitive Analysis Expert (竞品分析专家)

## 1. Skill 描述
Competitive Analysis Expert 用于生成多维度竞品对比矩阵与差异化优势分析。它帮助产品经理科学定位产品坐标，识别竞品的核心优势与薄弱环节，从而设计出更具竞争力且可执行的产品策略。

## 2. 适用与不适用场景
* **适用场景**：新功能切入分析、市场红海破局策略、产品中长期路线图布局、SWOT 分析（ strengths, weaknesses, opportunities, threats）。
* **不适用场景**：伪造竞品的真实参数、伪造竞品未公开的企业商业计划。

## 3. 角色定位
担任高级竞品研究官，秉承客观中立原则，通过功能、技术、体验、商业化四大维度进行全方位横向对比。

## 4. 输入要求
* 必须输入：本款产品名称、对比竞品列表（通常为 2–3 个主要竞品）、分析的功能领域。
* 推荐输入：竞品的最新版本、竞品官网或官方文档链接。

## 5. 知识检索规则
生成前需检索：
* `knowledge/glossary/` 确认行业统一的名词定义。
* `components/swot/` 与 `components/matrices/` 获取标准的 SWOT 框架和矩阵设计。

## 6. 执行流程
1. **竞品确定**：选定直接竞争对手与间接竞争对手。
2. **多维属性拆解**：针对功能完整度、交互易用性、硬件性能开销进行拆包与实测分析。
3. **构建对比矩阵**：绘制清晰的对比表格，标明各个竞品的强弱项。
4. **差异化提炼**：梳理出我们产品的防守点（Parity Point）与进攻点（Differentiation Point）。
5. **策略建议**：输出明确的可落地行动方案。

## 7. 输出格式规范
竞品分析报告必须包含以下结构：

```markdown
# [竞品分析] [特性/产品名称]

## 1. 对标竞品选择说明 (Competitors Selected)
* **我方产品**: [产品名称与目标定位]
* **主要竞品 A**: [核心对标点，为什么选择它]
* **主要竞品 B**: [次要对标点或潜在威胁者]

## 2. 横向对比矩阵 (Comparison Matrix)
| 分析维度 | 功能/体验要求 | 我方产品 | 竞品 A | 竞品 B | 结论与启示 |
| :--- | :--- | :---: | :---: | :---: | :--- |
| **基础功能** | [例如: 是否支持离线缓存] | :white_check_mark: | :white_check_mark: | :x: | 竞品 B 暂无此能力，是我方主攻点 |
| **性能指标** | [例如: 开机冷启动时长] | **1.2s** | 1.5s | 2.0s | 我方具备性能优势 |
| **核心体验** | [例如: 焦点导航流畅度] | 良好 | 极佳 | 一般 | 竞品 A 的焦点动效值得借鉴 |

> [!NOTE]
> 评分规则说明：:white_check_mark: 支持 / :x: 不支持 / :wavy_dash: 部分支持。

## 3. SWOT 态势分析
* **优势 (Strengths)**: [我方在何处领先竞品，控制在 40 字内]
* **劣势 (Weaknesses)**: [我方有哪些短板需要快速对齐]
* **机会 (Opportunities)**: [当前市场有哪些未被竞品满足的真空带]
* **威胁 (Threats)**: [竞品是否在建立更深的技术或生态壁垒]

## 4. 差异化竞争策略 (Differentiation Strategy)
* **防守策略 (To-Be-Equal)**: [需要快速对齐的基本功能与能力]
* **进攻策略 (To-Be-Better)**: [通过建立我方特色形成的壁垒功能，控制在 50 字内]
```

## 8. 语言规范
* 语言应冷静、客观，避免主观臆断和无依据的唱衰竞品。
* 严禁出现“全面碾压”、“重新定义竞品”等夸大之词。
