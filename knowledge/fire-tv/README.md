# 知识库 - Fire TV 开发与设计限制

* **版本状态**: Active
* **适用范围**: 搭载亚马逊 Fire OS 系统的电视与电视棒 (Fire TV Stick)
* **数据来源**: 亚马逊开发者门户 Fire TV Design Guidelines
* **更新时间**: 2026-07-11

## 核心定义
**Fire TV** 是亚马逊基于开源 Android（AOSP）二次开发的智能电视系统平台（Fire OS）。由于去除了谷歌 GMS 包，其推荐算法、支付系统与语音助手（Alexa）均深度绑定亚马逊生态。

## 核心交互与设计限制

1. **Launcher 主页深度定制限制**：
   - 亚马逊严控主屏（Launcher）的排版，厂商（OEM）无法自定义主屏的 Banner 推荐位，仅能在预留的“合作伙伴推荐”中呈现。
2. **语音助手 Alexa 独占性**：
   - 系统级语音识别必须由 Alexa 独占。第三方应用不得后台常驻监听语音热词。
3. **沉浸式横幅广告与背景静音要求**：
   - 当焦点移至主屏横幅推荐（Hero Row）时，通常会启动视频预览。规范要求在视频启动的 **前 3 秒内必须保持静音**，第 4 秒起才可淡入声音，以防突然啸叫惊扰用户。
4. **Android API 兼容性版本差异**：
   - Fire OS 7 基于 Android 9 (API 28)，Fire OS 8 基于 Android 11 (API 30)。由于 API 版本相对滞后，编写 TV 应用时需注意向后兼容性，避免直接调用 Android 12+ 的新系统类。
