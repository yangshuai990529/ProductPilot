# 知识库 - 电视系统菜单树层级设计规范

* **版本状态**: Active
* **适用范围**: 智能电视系统设置、快捷菜单 (Quick Menu) 及应用层级划分
* **数据来源**: 电视交互人因工程研究、主流 TV 菜单实测
* **更新时间**: 2026-07-11

## 核心定义
**菜单树（Menu Tree）** 是指电视系统中设置项与导航条目的逻辑分支结构。合理的菜单树设计可帮助用户快速定位功能，避免在大屏视距下迷失在复杂的迷宫化层级中。

## 菜单树三大黄金原则

1. **扁平化设计 (不超过 3 层)**：
   - **L1 (一级主菜单)**: 侧边栏导航（如：图像、声音、网络、系统）。
   - **L2 (二级子分类)**: 逻辑卡片列表（如：图像模式、高级设置）。
   - **L3 (三级叶子节点)**: 最终调节控件（如：亮度进度条滑块、开关切换器）。
   - 禁止出现 L4 级及更深层次的跳转，必要时使用弹窗（Dialog）或抽屉（Drawer）进行扁平化降级。
2. **快捷操作覆盖 (Quick Menu)**：
   - 电视在任何信号源播放时，按下遥控器 [设置/菜单] 键，必须弹出不遮挡核心画面的悬浮快捷菜单（底置或侧置，占比 <= 25% 宽度）。
   - 快捷菜单提供用户最高频的调节（图像模式、声音模式、定时关机、输入源切换），无需中断用户观影。
3. **焦点环绕机制 (Wrap-around)**：
   - 当用户在菜单列表底部按下 **下键** 时，焦点不应被锁死，而应重置跳转到该列表的首行（环绕），以缩短翻页路径。

## 典型电视设置菜单树结构

```
[设置 (Home Settings)]
 ├── [图像 (Picture)]
 │    ├── [图像模式 (SDR/HDR/DV)] -> 切换 [标准/电影/游戏/体育]
 │    ├── [亮度 (Brightness)] -> 弹出滑块 (0-100)
 │    └── [高级画质 (Advanced)] -> 展开子项 [Local Dimming / MEMC / 降噪]
 ├── [网络 (Network)]
 │    ├── [无线网络 (Wi-Fi)] -> 展示可用 SSID 列表
 │    └── [有线网络 (Ethernet)] -> 展示 IP 地址与连接状态
 └── [关于 (About)]
      ├── [系统更新 (System Update)]
      └── [设备状态 (Status)]
```
