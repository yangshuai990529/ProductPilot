# AI Agent 行为规则 (AGENTS.md)

本文件定义了在 ProductPilot 项目中运行或为其生成内容的 AI Agent 的系统级指令与行为边界。无论是本项目的贡献 Agent，还是 ProductPilot 内部的多 Agent 评审引擎，都必须严格遵守本规范。

## 1. 语言与文风规则

* **简体中文默认**：除非用户明确要求英文，否则所有生成的文档、PRD、PPT 文案、页面说明、错误提示和代码注释均默认使用**简体中文**。
* **专业术语规范**：专业英文术语首次出现时，必须采用 `中文名称（English Name）` 的格式。
  * 例如：产品需求文档（Product Requirements Document，PRD）、系统需求（System Requirement，SR）。
* **拒绝营销词汇**：文案必须使用客观、专业、可测试的产品语言。禁止使用空洞无物的营销词汇，如“赋能”、“引领未来”、“颠覆创新”等。
* **简洁高效**：结论明确、逻辑完整、避免大段文字，能用图表/对比矩阵表达的绝不用纯文本。

## 2. 文字层级规范 (L1 - L6)

AI Agent 必须能够清楚识别并严格区分文字层级，通过字号、字重、位置和间距体现其层级，禁止仅靠颜色区分层级：

* **L1：页面主标题**：控制在 6–18 个中文字符，表达核心主题，不使用完整长句。
* **L2：页面副标题/核心结论**：控制在 20–40 个中文字符，使用一句完整的话表达本页的核心判断。
* **L3：模块标题**：用于区分不同内容模块（例如：当前现状、问题原因、影响范围、产品方案、用户价值）。
* **L4：功能点/关键要点**：控制在 12–25 个中文字符，每条表达一个独立的、可开发或可测试的信息。
* **L5：辅助说明**：补充条件、限制、备注等，字号明显小于正文，仅在必要时使用。
* **L6：页脚/保密信息**：包括数据来源、文档版本、更新时间、保密标识等。

## 3. 跨应用与软件兼容性原则

* **通用标准优先**：不依赖单个办公/设计软件的私有特效（如 Keynote 的特定过渡动画、PowerPoint 专属 SmartArt）。
* **可编辑性**：生成的 PPTX、原型、表格和流程图必须是可编辑的文本或形状，禁止将表格/流程图截图输出。
* **相对路径**：网页和原型中的静态资源必须使用相对路径，支持本地零配置运行或 GitHub Pages 部署。

## 4. 评审引擎工作流 (Review Engine)

任何产品方案、PRD 或系统需求生成后，AI Agent 必须执行自动 Review 流程，涵盖以下角色维度：
1. **产品经理视角**：问题是否真实、产品价值是否明确、验收标准是否完整。
2. **UX 设计视角**：界面交互是否符合十英尺原则（对于电视端）、入口与路径是否合理。
3. **研发/架构师视角**：开发依赖是否明确、时序关系是否完整、是否存在性能瓶颈。
4. **测试视角**：需求是否具备可测试性，边界条件是否闭环。
5. **法务与隐私视角**：是否存在用户数据出境、合规性风险。

评审结果应采用 markdown 列表形式输出，并使用明确的级别标记：`Blocker` / `High` / `Medium` / `Low`。
