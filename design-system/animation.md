# 设计系统 - 动效设计规范 (Animation)

* **更新时间**: 2026-07-11

在智能电视大屏端（10-foot UI）及网页原型中，平滑的动效不仅能提供愉悦的视觉享受，更能为遥控器十字键操作提供清晰的焦点转移暗示。

## 1. 电视端焦点移动微动效 (TV Focus Animation)

* **放大特效 (Scale Up)**:
  - 获得焦点（Focused）的卡片进行中心等比例放大，放大倍率控制在 **1.05 至 1.08** 倍。
  - **红线约束**：禁止放大比例超过 1.15，否则卡片内容会发生重叠或导致大屏边缘视觉溢出。
* **弹性曲线 (Easing Curve)**:
  - 焦点移动动画时间统一使用 **200 毫秒至 250 毫秒** (ms)。
  - 缓动函数（Easing Function）推荐采用标准贝塞尔曲线：
    `cubic-bezier(0.25, 1, 0.5, 1)` (OutQuad) - 呈现出轻快、无延迟的移动触感。

---

## 2. 跨应用动画降级策略
* **演示文稿导出**:
  - 导出为标准 PPTX 幻灯片时，必须彻底去除所有仅在 Keynote 或 PowerPoint 中支持的闭源高级特效（如 Keynote 的神奇移动 Magic Move、PPT 专属的复杂 SmartArt 变形）。
  - 使用最基础的“**淡入淡出 (Fade)**”或“**推入 (Push)**”作为跨软件演示的安全默认过渡动画，确保在 Keynote, WPS, PowerPoint 中均有正常流畅的翻页效果。
* **网页原型端**:
  - 原型动画使用纯 CSS Transitions 实现，不依赖复杂的外部 JS 动画库（如 GSAP），保证弱网下本地双击运行的性能流畅。
