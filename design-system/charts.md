# 设计系统 - 图表与流程图表达规范 (Charts)

* **更新时间**: 2026-07-11

在产品设计中，图表和关系图的选型必须严格对应数据逻辑与信息层级。严禁为了好看而滥用不相关的图表。

## 1. 业务逻辑与图表选型对照表

| 表达逻辑 | 推荐的图示表达方式 | 组件库参考 | 说明 |
| :--- | :--- | :--- | :--- |
| 跨团队协作 / 页面流转 | **泳道流程图 (Swimlane Flowchart)** | `flowcharts/` | 明确划分 Client, Server, Driver 职责边界。 |
| 系统间调用时序 | **时序图 (Sequence Diagram)** | `sequence-diagram/` | 展现同步/异步交互及接口握手包传递。 |
| 阶段性交付排期 | **甘特图 / 时间轴 (Gantt/Timeline)** | `timelines/` | 粗颗粒度表达 MVP, V1, V2 发版里程碑。 |
| 现状痛点因果分析 | **“现象—原因—影响”因果链** | `matrices/` | 直观还原技术故障或用户痛点的底层传导路径。 |
| 竞品功能多维对比 | **对比矩阵 (Comparison Matrix)** | `matrices/` | 以二维表格配合状态图标，呈现强弱态势。 |
| 复杂操作状态转换 | **状态机图 (State Diagram)** | `state-diagram/` | 描述 Wi-Fi/DFS 避让、焦点状态机变迁。 |

---

## 2. 流程图编写原则 (Flowcharts)
* **单入口与单出口**：所有流程图必须有明确的 `Start` 节点，并且逻辑必须闭环，有对应的终点。
* **异常分支判定**：每一个判定框（Decision Diamond）必须延伸出至少两条路径，且必须标注 `Y` (Yes) 或 `N` (No)。
* **可编辑性**：在 PPTX 中，流程图必须由原生形状与连接线（Connectors）拼接而成，不允许合并导出为无法修改的 PNG 位图。
