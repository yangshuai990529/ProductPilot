# 设计系统 - 跨应用与平台兼容性规范 (Compatibility)

* **更新时间**: 2026-07-11

ProductPilot 生成的内容必须最大化兼容主流办公、设计和演示应用，拒绝闭源或私有技术绑定。

## 1. 跨软件导出基准规范

| 软件名称 | 支持格式 | 兼容策略 / 红线约束 |
| :--- | :--- | :--- |
| **MS PowerPoint** | `.pptx` (16:9) | 唯一核心分发源。禁止使用 Macro (宏) 与 ActiveX 控件。 |
| **Apple Keynote** | 导入 `.pptx` | 确保字体回退顺畅，禁止使用 Keynote 专属 Magic Move。 |
| **WPS 演示** | 打开 `.pptx` | 不可使用 WPS 私有云组件，保证在 Windows/Linux 端无错位。 |
| **Figma / MasterGo** | SVG / PDF 导入 | 所有的图标与原型组件必须是可编辑形状，禁止截图代替。 |
| **Web 浏览器** | 标准 HTML/CSS/JS | 兼容 Chrome, Safari, Edge, Firefox。静态资源使用相对路径。 |

---

## 2. 导出文件完整性自检清单 (Pre-flight Checklist)
任何由 ProductPilot 生成并输出的文件，在交付给用户前必须自动或手动通过以下清单核对：

* [ ] **字体替换排查**：在无 PingFang SC 字体环境（如纯 Windows 机器）下打开，检查排版有无折行、文本框是否因字宽变化导致文字溢出（Overflow）。
* [ ] **图片完整性**：所有插入的图片必须为嵌入式（Embedded），禁止使用只有本地硬盘绝对路径才可读取的外链图片（防止分发后图片丢失）。
* [ ] **圆角与阴影表现**：检查可编辑卡片的圆角（如 8px）、投影（Box Shadow）在不同 Office 版本下是否失效或变成直角黑影。
* [ ] **连接线锁定**：流程图中的连线必须绑定在形状的锚点（Anchors）上，以便用户后续拖拽修改时，连线不会断裂或移位。
* [ ] **表格非截图**：PPTX 中的对比矩阵、数据明细必须为可编辑的表格单元格，绝对禁止使用网页截图粘贴。
