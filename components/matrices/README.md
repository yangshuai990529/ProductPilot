# 组件 - 对比与痛点矩阵 (Matrices)

本组件用于产品立项方案中分析现状因果链或横向对比竞品。

## 1. “现象—原因—影响” 因果链模板

```
[ 现象 (Symptom) ]
 └─> 电视开机冷启动时间长达 45 秒 (用户流失率增加 12%)
      
      [ 原因 (Cause) ]
       └─> 系统启动时，同步加载了 15 个非核心后台常驻服务
            
            [ 影响 (Impact) ]
             └─> 内存溢出 (OOM)，低端机型高概率卡死崩溃
```

## 2. 竞品对比矩阵模板
参阅 `templates/competitive-analysis/README.md` 中的表格定义。

## 3. 设计规范
* 结构清晰，逻辑必须是一对一传导。
* 因果分析需给出具体的量化负面影响（如：流失率增加 12%）。
