# 组件 - 用户旅程图 (User Journey Map)

本组件用于在用户研究报告中直观描述用户的痛点分布与情绪曲线。

## 1. 用户旅程矩阵模板

| 步骤阶段 | 1. 查找内容 (发现) | 2. 扫码购买 (转化) | 3. 开通使用 (体验) |
| :--- | :--- | :--- | :--- |
| **用户动作** | 在电视主页翻看海报 | 使用手机微信扫码支付 | 电视弹出购买成功提示 |
| **情绪指数** | :neutral_face: 平稳 (查找繁琐) | :frowning: 焦躁 (扫码总失败) | :smiley: 喜悦 (瞬间解锁) |
| **用户痛点** | 分类不清晰，层级过深 | 电视端二维码过期或模糊 | 解锁有时延迟，需重开应用 |
| **产品机会** | 优化导航树，不超过3层 | 缩短刷新间隔，提高对比度 | 引入 Webhook 实时通道推送 |

## 2. 设计规范
* 情绪指数采用 emoji（如 :smiley: / :frowning:）进行可视化表达。
* 产品机会点需与前面的痛点一一对应。
