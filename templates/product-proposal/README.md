# 模板 - 产品立项方案模板 (Product Proposal Template)

## 1. 使用场景
本模板适用于业务新功能开发立项、跨团队产品架构重大调整以及中型产品特性的立项评审会（Product Review Board）。

## 2. 页面与章节结构

### 2.1 必选页面/章节
1. **背景与核心痛点 (Background & Painpoint)**:
   - 描述业务现状。通过“现象 → 原因 → 影响”的因果链，证明问题真实存在。
2. **产品方案设计 (Product Solution)**:
   - 核心功能框架，用户体验核心路径图（User Journey / Flow）。
3. **用户与业务价值 (Value)**:
   - 该功能上线能带来什么可测量的成功指标（KPI/Northstar Metric）。
4. **功能路线图 (Roadmap)**:
   - 阶段性的发版规划（MVP -> V1.0）。

### 2.2 可选页面/章节
- 竞品对比、备选方案（Alternative Solutions）分析。

## 3. 推荐表达图表
- 使用 **“现象—原因—影响”因果链** 描述背景。
- 使用 **泳道流程图** 描述跨系统业务流程。
- 使用 **价值卡片** 展现上线后的用户收益。

## 4. 输出检查列表 (Checklist)
* [ ] 是否写明了项目成功的量化指标？
* [ ] 对开发团队的依赖（如：是否有硬件限制）是否指明？
* [ ] 异常流程是否已在流程图中闭环？
