# 模板 - 周报与标准作业程序模板 (Weekly Report & SOP Template)

## 1. 使用场景
本模板适用于产品团队进行每周工作总结汇报（Weekly Report），以及规范团队日常标准作业程序（Standard Operating Procedure，SOP）。

## 2. 页面与章节结构

### 2.1 周报结构 (Weekly Report)
1. **本周工作进展与成果 (Key Achievements)**:
   - 已上线功能、已完成的评审、重大问题的解决。
2. **核心业务指标大盘 (Metric Dashboard)**:
   - 关注活跃用户、崩溃率、转化率等核心数据环比变化。
3. **下周重点工作规划 (Next Steps)**:
   - 拟开展的项目、计划完成的文档。
4. **当前面临的问题与风险 (Risks & Issues)**:
   - 需要协调的资源、有延期风险的项目（标明红黄绿状态）。

### 2.2 SOP 结构
1. **目的与应用范围 (Objective)**:
   - 规定此工作流的触发原因。
2. **标准步骤说明 (Standard Steps)**:
   - 1-2-3 顺序的具体执行指南。
3. **输出结果与验证标准 (Output Validation)**:
   - 执行完 SOP 后的产物检查。

## 3. 推荐表达图表
- 使用 **指标升降箭头表格**（对比上周数据）。
- 使用 **红黄绿三色卡片**（代表项目健康状态）。

## 4. 输出检查列表 (Checklist)
* [ ] 周报内容是否控制在一屏内，不出现冗长的过程描述？
* [ ] 每一个风险点是否都匹配了明确的认领负责人（Owner）与解决时限？
* [ ] SOP 中的操作步骤是否可完全被新成员照搬执行？
