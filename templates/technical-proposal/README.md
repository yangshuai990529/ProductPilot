# 模板 - 技术对接与系统需求模板 (Technical Proposal Template)

## 1. 使用场景
本模板适用于前后端开发团队对接、跨系统接口定义、硬件驱动与底层应用握手机制规划，承接产品需求并输出系统需求（SR）。

## 2. 页面与章节结构

### 2.1 必选页面/章节
1. **系统拓扑结构 (System Topology)**:
   - 说明数据在各层（UI、网关、中台服务、数据库、第三方服务）的物理和逻辑走向。
2. **详细时序逻辑流程 (Sequence Flow)**:
   - 各个系统服务之间接口调用的先后时序，数据校验节点及分支走向。
3. **接口契约定义 (API Contract)**:
   - 规定请求 URL、Method、Headers、参数类型及含义、错误码与对应响应体。
4. **非功能性指标约束 (Non-functional Requirements)**:
   - 核心接口 QPS、最大允许响应时延、敏感数据传输加密手段。

## 3. 推荐表达图表
- 使用 **Mermaid 架构图**。
- 使用 **Mermaid 时序图**。
- 使用 **数据流泳道图**。

## 4. 输出检查列表 (Checklist)
* [ ] 每一个接口参数的数据类型（如 String / Integer）是否明确标注？
* [ ] 是否明确排查了所有网络中断、鉴权失败等异常处理逻辑？
* [ ] 涉及敏感字段（如 Token、密码）是否有安全加密传输要求？
