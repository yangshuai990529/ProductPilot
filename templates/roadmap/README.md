# 模板 - 产品路线图模板 (Roadmap Template)

## 1. 使用场景
本模板适用于向研发、测试、运营和市场团队宣讲中长期的产品版本排期，以及大型特性交付的关键节点对齐。

## 2. 页面与章节结构

### 2.1 必选页面/章节
1. **中长期愿景与战略聚焦 (Strategic Focus)**:
   - 路线图的北极星指标（如：提升系统稳定性至 99.9%）。
2. **三阶段横向时间轴路线图 (Roadmap Timeline)**:
   - 近期（MVP/Alpha）交付范围。
   - 中期（V1.0/Beta）功能规划。
   - 远期（V2.0/规划中）演进设想。
3. **跨团队研发依赖关系 (Cross-team Dependencies)**:
   - 前端、后台、算法、硬件的交付对齐时序。

## 3. 推荐表达图表
- 使用 **甘特图 (Gantt Chart)** 或 **横向阶段泳道图** 表达时间排期。
- 使用 **卡片连线** 表达关键依赖路径。

## 4. 输出检查列表 (Checklist)
* [ ] 里程碑是否明确关联了具体的业务交付物，而非空泛的目标？
* [ ] 是否包含了跨团队的底层物理依赖？
* [ ] 时间节点是否具备合理的缓冲时间（Buffer）？
